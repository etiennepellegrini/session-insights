# Session Insights

Observations from Claude Code sessions — interaction patterns, workflow
preferences, and technical learnings that aren't captured elsewhere in
the memory files. Written by Claude (via `/reflect`);
curated by you (via `/curate` or manual editing).

**Format for new entries:**
- Use `## YYYY-MM-DD` date headers; reuse today's if it exists
- One insight per line: `- [tag] Observation text`
- Tags: `[workflow]` `[communication]` `[technical]` `[preference]` `[blindspot]`
- Write about *interaction dynamics*, not task summaries
- Be specific and actionable ("prefers X over Y in context Z")
- Don't duplicate what's already in CLAUDE.md
- If an insight contradicts something in CLAUDE.md, note it — user decides
